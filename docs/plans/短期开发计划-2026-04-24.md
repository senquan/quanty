## 📊 项目现状概览

**Quanty** 是一个基于 **FastAPI + Vue3(VBEN Admin)** 的企业级量化交易系统，核心模块包括：

- ✅ **用户/角色/菜单管理**（RBAC 权限体系）
- ✅ **量化回测引擎**（多数据源 + 策略执行）
- ✅ **技术指标系统**（30+ 指标）
- ✅ **高级性能分析**（50+ 指标）
- ✅ **模拟交易服务**（华泰证券模拟）
- ⚠️ **前端页面**（部分实现，部分待完善）

## 🔴 高优先级（立即修复 - Bug & 健壮性）

### H1-1: Async/Sync 数据库会话混用问题 ✅ 已修复
**影响**: `users.py`、`roles.py`、`quant.py` 使用同步 `Session`，但 `database.py` 创建的是 `AsyncSession`
- **文件**: `app/api/api_v1/endpoints/users.py`
- **文件**: `app/api/api_v1/endpoints/roles.py`
- **文件**: `app/api/api_v1/endpoints/quant.py`
- **修复**: 统一改为 `AsyncSession`，所有 `db.query()` 改为 `select() + await db.execute()`
- **变更**: `db.commit()` → `await db.commit()`，`db.refresh()` → `await db.refresh()`
- **状态**: ✅ 完成 (2026-04-24)

### H1-2: `user` 模型缺少 `is_active` 字段 ✅ 已修复
**影响**: `users.py` 中创建/更新用户时使用了 `user.is_active = is_active`，但 `User` 模型中该字段不存在
- **文件**: `app/models/user.py` (原: `status = Column(Integer, default=1)`)
- **文件**: `app/schemas/user.py` (Line 10: `is_active: bool = True`)
- **文件**: `app/api/api_v1/endpoints/auth.py` (Line 100: `current_user.status == 1`)
- **修复**: `status` 重命名为 `is_active`，auth.py 中 `current_user.status` → `current_user.is_active`
- **注意**: `is_active` 字段类型为 `Integer` (0=禁用/1=启用)，schema 传入 `bool` 端点需做 `bool→int` 转换
- **状态**: ✅ 完成 (2026-04-24)

### H1-3: `user` 模型缺少 `role_id` 外键 ✅ 已修复
**影响**: schemas 中有 `role_id`，但 model 中没有对应字段
- **文件**: `app/schemas/user.py` Line 11: `role_id: Optional[int] = None`
- **修复**: 在 `User` 模型添加 `role_id = Column(Integer, ForeignKey("roles.id"))`
- **状态**: ✅ 完成 (2026-04-24)

### H1-4: `users.py` 搜索条件引用不存在的 `full_name` 列 ✅ 已修复
**影响**: Line 26 使用 `User.full_name.contains(search)`，但模型中是 `realname`
- **文件**: `app/api/api_v1/endpoints/users.py`
- **修复**: `User.full_name` → `User.realname`
- **状态**: ✅ 完成 (2026-04-24)

### H1-5: `dict()` 方法 → `model_dump()` (Pydantic v2 兼容) ✅ 已修复
**影响**: 3 个 endpoint 文件使用 `dict(exclude_unset=True)`，Pydantic v2 中已废弃但仍可工作
- **文件**: `app/api/api_v1/endpoints/users.py` → `model_dump()`
- **文件**: `app/api/api_v1/endpoints/roles.py` → `model_dump()`
- **文件**: `app/api/api_v1/endpoints/menus.py` → `model_dump()`
- **状态**: ✅ 完成 (2026-04-24)

### H1-6: 状态字段命名统一 ✅ 已修复
**影响**: 角色用 `is_active`，菜单用 `is_enabled`，用户用 `is_active`（原是 `status`）
- **修复**: 统一各端点使用模型实际字段名
  - users.py: `user.is_active` ✅
  - roles.py: `role.is_active` ✅
  - menus.py: `menu.is_enabled` ✅ (保持语义一致：菜单是 enabled/disabled，用户/角色是 active/inactive)
- **状态**: ✅ 完成 (2026-04-24)

**附加修复** (排查中发现):
- `roles.py` `delete_role`: `db.query(User).filter(User.role_id == role_id)` → `select(User).where(User.role_id == role_id)` (Async)
- `auth.py` `/me` 端点: `current_user.status` → `current_user.is_active` (已记录在上)
- `users.py` `create_user`: 添加 `is_active` bool→int 转换 (`1 if is_active else 0`)
- `users.py` `update_user`: 添加 `is_active` bool→int 转换 + `full_name` → `realname` 映射
- 所有 endpoint 返回 `__dict__` 时清理 `_sa_instance_state` 内部属性

## 🟡 中优先级（核心功能增强）

### M1: 策略 CRUD 增强（POST/PUT/DELETE 策略）
**现状**: 只有 `create_strategy` (POST) 和查询，缺少 update/delete
- **文件**: `app/api/api_v1/endpoints/quant.py`
- **新增**:
  - `PUT /api/v1/quant/strategies/{id}` - 更新策略（更新时重新验证代码）
  - `DELETE /api/v1/quant/strategies/{id}` - 删除策略
- **风险**: 低 | **耗时**: 0.5 天

### M2: 回测任务异步化支持
**现状**: 回测同步执行，长回测会阻塞请求
- **文件**: `app/api/api_v1/endpoints/quant.py`
- **方案**: 
  1. 引入 Celery/ARQ 异步任务队列
  2. 或返回 task_id + 轮询查询结果 (`/api/v1/quant/backtest-status/{task_id}`)
- **文件**: 需新增 `app/services/celery_tasks.py` 或 `app/services/tasks.py`
- **风险**: 中 | **耗时**: 1.5 天

### M3: 数据源模块扩展 - A股/港股支持
**现状**: 仅支持 Yahoo Finance（美股）和 CCXT（加密货币）
- **新增数据源**:
  - AkShare（A股市场数据）
  - Tushare（A股市场数据）
- **新增数据源**:
  - 港股可通过 Yahoo Finance (如 `0700.HK`)
- **文件**: `app/services/data_sources.py`
- **风险**: 中 | **耗时**: 2 天

### M4: 策略沙箱执行优化
**现状**: `exec(strategy_code, strategy_globals)` 直接执行，危险函数检查较粗
- **改进**:
  1. 使用 `ast` 模块做更精细的语法树检查
  2. 捕获 `SyntaxError`, `TypeError`, `AttributeError` 等异常
  3. 添加执行超时（`signal.alarm` 或 `concurrent.futures.ThreadPoolExecutor`）
  4. 支持多个信号量的 `buy`/`sell`（按 index 触发，而非全局）
- **文件**: `app/services/backtest_engine.py`
- **风险**: 中 | **耗时**: 1 天

### M5: 回测结果持久化 - 交易明细表
**现状**: 回测结果仅存储汇总指标，缺失交易明细
- **新增模型**: `BacktestTrade` 表（order_id, strategy_id, type, price, quantity, timestamp, pnl, commission）
- **修改**: `BacktestEngine` 返回详细交易记录，`BacktestResult` API 返回完整信息
- **文件**: `app/models/quant.py`, `app/services/backtest_engine.py`, `app/api/api_v1/endpoints/quant.py`
- **风险**: 低 | **耗时**: 1 天
