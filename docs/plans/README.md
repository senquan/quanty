# Plans - AI 每日任务目录

本目录由 AI Agent 自动管理，用于安排和记录每日开发任务。

## 📋 目录结构

```
plans/
├── README.md                  # 本说明文件
├── 2026-04-14.task.md          # 2026年4月14日任务清单
├── 2026-04-14.report.md        # 2026年4月14日任务总结
├── 2026-04-15.task.md          # 2026年4月15日任务清单
├── 2026-04-15.report.md        # 2026年4月15日任务总结
└── ...
```

## 🤖 工作流程

### 1. 每日任务生成
AI Agent 每天自动（或根据用户指令）生成当日任务文件：
- **文件名格式**: `YYYY-MM-DD.task.md`
- **内容**: 当日需要完成的任务清单

### 2. 任务执行
AI Agent 按照任务文件逐项执行：
- 执行过程中的状态实时更新到任务文件
- 每完成一项，标记 `[x]` 状态

### 3. 任务总结
任务全部完成后，AI Agent 生成当日报告文件：
- **文件名格式**: `YYYY-MM-DD.report.md`
- **内容**: 任务完成情况、遇到的问题、解决方案、代码变更摘要

## 📝 任务文件格式

```markdown
# YYYY-MM-DD 任务清单

**状态**: 进行中 / 已完成 / 部分完成

## 任务列表

- [ ] 任务1：描述
- [ ] 任务2：描述
- [x] 任务3：描述（已完成）

## 优先级说明

- 🔴 高优先级
- 🟡 中优先级
- 🟢 低优先级
```

## 📊 报告文件格式

```markdown
# YYYY-MM-DD 任务报告

**日期**: YYYY-MM-DD
**状态**: 已完成 / 部分完成

## 完成的任务

1. ✅ 任务1：描述
   - 具体操作说明
   - 相关文件变更

2. ❌ 未完成的任务：描述
   - 原因说明
   - 下一步计划

## 代码变更摘要

- 新增文件: xxx
- 修改文件: xxx
- Git commit: xxx

## 遇到的问题与解决

- 问题描述
- 解决方案

## 明日计划

- [ ] 计划1
- [ ] 计划2
```

## ⚙️ 管理命令

AI Agent 可通过以下方式管理任务：

```bash
# 创建今日任务
> 请为我创建今天的任务清单

# 查看任务状态
> 查看今天的任务进度

# 强制完成任务
> 完成今天的任务并生成报告

# 手动指定任务
> 今天的任务是：1.修复bug 2.添加新功能
```
